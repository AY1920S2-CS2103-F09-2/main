About the project

My team of 4 software engineering students and I were tasked with enhancing a basic command line interface desktop address book application for our Software Engineering project. We chose to morph it into a task management system called JelphaBot. This enhanced application enables university students to file and recall task data, manage task scheduling and view productivity levels all in one place.

Summary of contributions

This section shows a summary of my coding, documentation, and other helpful contributions to the team project.

Enhancement added:

I added the start and stop a timer feature for tasks as well as a viewing panel for the user’s overall productivity for the day. I also incorporated the command line history.

  • What it does: The start <index> command allows the user to start a timer for the task specified by the index provided. The user may stop this timer with the stop <index> command. If the task timed is due today or in the next seven days, the productivity panel would show the total time spent on that task. The panel also shows the completion rate of tasks due today. To retrieve the command history, the UP-arrow key retrieves the command entered right before the one entered. The DOWN-arrow key does the opposite, meaning that it retrieves the command entered after the one entered if any.

  • Justification: In the event the user wants to gauge the amount of time they have spent on their tasks and how many tasks they have completed or an overview of their productivity in that day, it is easy to view all this information in one tab. To reduce the hassle of having to key in similar commands repeatedly, users can easily retrieve past commands and enter it.

  • Highlights: This enhancement works with existing as well as future commands. An in-depth analysis of design alternatives was necessary to decide the layout of the productivity panel and how effective it was in improving overall user experience of the application. The implementation was also challenging because it was difficult to decide how much data to present to users. Out of all the data stored, I had to decide which calculations are the most beneficial for the user and would boost their productivity and efficiency in their tasks.

  • Credits: Stop and Start command classes were adapted from existing commands such as Edit command. Command line history was adapted from existing implementations. Code contributed: [Productivity UI] [Productivity model] [Start Timer] [Stop Timer] [Productivity Test Code] [Json Test]
